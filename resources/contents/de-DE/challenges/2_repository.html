Erstelle ein neues Repository auf deinem Computer.

Repositories

Ein Repository ist eine Sammlung verwandter Elemente. In unserem Fall, beim Schreiben von Software, ist es eine Sammlung von Dateien, die zu einem Software-Projekt gehören. Du kannst es dir als Projektordner mit allen relevanten Dateien darin vorstellen. Tatsächlich wird es so auf deinem Computer aussehen. Manchmal werden sie kurz "Repos" genannt.

Du sagst Git, was dein Projekt ist, und Git wird anfangen, alle Änderungen an diesem Ordner zu verfolgen. Das macht es zu einem Git-Repository: ein Ordner mit Elementen, die von Git verfolgt werden. Git verfolgt, wann Dateien hinzugefügt, entfernt oder sogar ein einzelner Buchstabe in einer einzelnen Datei geändert wird. All dies plus wer es gemacht hat und wann wird von Git verfolgt. In der Software wird das Verfolgen von Änderungen wie diese Versionskontrolle genannt.

Terminal verwenden

Terminal (oder Bash) ist eine Möglichkeit, deinem Computer Anweisungen zu geben, indem du Befehle tippst, anstatt Anwendungen mit deiner Maus zu klicken. Du kannst Dateien umbenennen, Dateien öffnen, neue Ordner erstellen, zwischen Verzeichnissen (Ordnern) wechseln und mehr, alles durch das Ausführen getippter Befehle. Du kannst sogar einen Texteditor für Code (wie Vim) in deinem Terminal verwenden und musst nie verlassen!

Neben der Navigation auf deinem Computer kannst du auch Programme im Terminal verwenden, die eine Befehlszeilen- Oberfläche (CLI) haben, was bedeutet, dass sie mit Befehlen im Terminal ausgeführt werden können. Git ist eines davon. Der erste Teil des Befehls lässt deinen Computer wissen, dass du mit Git sprichst. Die Teile danach sind die Befehle und die verschiedenen Optionen, auf die Git reagieren soll.

In Git-it lernst du einige grundlegende Befehlszeilen-Aktionen (zusätzlich zu Git-Befehlen) für die Navigation auf deinem Computer; sie werden in den Schritten beschrieben.

Ein Repository erstellen

Du wirst einen neuen Projektordner erstellen und ihn dann als Git-Repository initialisieren. Wir nennen den Ordner 'hello-world' in dieser Challenge, aber du kannst einen anderen Namen wählen, wenn du bereits einen Ordner mit diesem Namen hast (es ist ein üblicher Name für erste Projekte).

Zeit, dein Terminal zu öffnen!

In deinem Terminal-Fenster tippe diese Befehle, einen nach dem anderen, und drücke nach jedem Enter/Return.

Erst, erstelle einen neuen Ordner:
Tipp: mkdir steht für make directory

mkdir hello-world

Dann gehe in diesen Ordner:
Tipp: cd steht für change directory

cd hello-world

Schließlich, sage Git, den Ordner zu initialisieren (mit der Verfolgung zu beginnen), in dem du dich jetzt befindest:

git init

Der letzte Befehl sollte etwas zurückgeben, das mit "Initialized empty Git repository" beginnt. Die anderen Befehle geben nichts zurück.

Du hast es geschafft! Wenn du doppelt sicher sein möchtest, dass es ein Git-Repository ist, tippe git status und wenn es nicht 'fatal: Not a git repository...' zurückgibt, bist du goldrichtig!
