Forke ein Projekt von GitHub.com und klone es lokal.

Forks

Bisher hast du ein Projekt lokal erstellt und zu GitHub gepusht, aber das ist nur die Hälfte des Spaßes! Die andere Hälfte ist die Zusammenarbeit mit anderen Menschen und anderen Projekten.

Wenn du ein Repository forkst, erstellst du eine Kopie davon in deinem GitHub-Konto. Deine geforkte Kopie beginnt ihr Leben als Remote-Repository—es existiert nur in deinem GitHub-Konto, nicht auf deinem Computer. Forks werden verwendet, um deine eigene Version eines Projekts zu erstellen (diese Abweichung vom Original ist wie einen Abzweig auf der Straße zu nehmen) oder deine Änderungen (wie Bugfixes oder neue Features) an das ursprüngliche Projekt zurückzugeben.

Um ein geforktes Repository von deinem GitHub-Konto auf deinen Computer zu bekommen, klonst du es. Diese Klon-Aktion kopiert das Remote-Repository auf deinen Computer, damit du lokal daran arbeiten kannst.

Das Patchwork-Repository forken

Das Projekt, mit dem wir arbeiten werden, ist unter github.com/jlord/patchwork. Gehe zu dieser Seite und klicke auf den 'Fork' Button oben rechts. Sobald die Fork-Animation abgeschlossen ist, hast du eine Kopie in deinem Konto. Kopiere die HTTP-URL deines Forks aus der Adressleiste deines Browsers, das ist die Adresse deines Forks auf GitHub's Servern.

Deinen Fork lokal klonen

Jetzt wirst du im Terminal das Repository auf deinen Computer klonen. Es wird automatisch einen neuen Ordner für das Repository erstellen, daher musst du keinen selbst erstellen. Aber stelle sicher, dass du es nicht innerhalb eines anderen Git-Repository-Ordners klonst!

Stelle also sicher, dass du dich nicht innerhalb des 'hello-world' Verzeichnisses aus den vorherigen Challenges befindest. Um zurückzugehen und diesen Ordner zu verlassen, damit du klonen kannst, folge diesen Schritten:

Gehe aus deinem 'hello-world' Ordner zurück:
Tipp: die zwei Punkte bedeuten, einen Verzeichnislevel zurückzugehen

cd ..

Jetzt, da du nicht mehr in einem anderen Git-Repository bist, klone deinen Fork:

git clone <URLFROMGITHUB>

Gehe in den Ordner, den es für deine lokale Kopie des Forks erstellt hat (in diesem Fall 'patchwork' genannt).

cd patchwork

Jetzt hast du eine Kopie des Repositories auf deinem Computer und es ist automatisch mit dem Remote Repository (deiner geforkten Kopie) in deinem GitHub-Konto verbunden. Tippe git remote -v, um zu sehen, dass die Adresse zum Fork bereits eingerichtet ist.

Mit dem ursprünglichen Repository verbinden

Was ist, wenn sich das ursprüngliche Repository, das du geforkt hast, ändert? Du möchtest in der Lage sein, diese Änderungen auch zu pullen. Lass uns also eine weitere Remote-Verbindung hinzufügen, diesmal zum ursprünglichen github.com/jlord/patchwork Repository mit seiner URL.

Du kannst diese Remote-Verbindung beliebig benennen, aber typischerweise verwenden Leute den Namen 'upstream'; lass uns das hier verwenden.

git remote add upstream https://github.com/jlord/patchwork.git

Um sicherzugehen, dass du die korrekten Remotes eingerichtet hast, tippe git remote -v, um die Adressen aufzulisten, die du gespeichert hast. Du solltest ein 'origin' Remote mit der Adresse deines Forks haben und dann ein 'upstream' Remote mit der Adresse zum ursprünglichen, der URL, die oben in diesem Schritt notiert ist.
