Merge deinen Branch lokal, lösche den Branch und pull vom Upstream.

Aufräumen

Sobald du Branches nicht mehr brauchst, kannst du sie lokal und remote löschen. Das ist hilfreich, damit du nicht mit einem Haufen alter Branches endest.

Lokal mergen

Dein Pull Request wurde gemergt! Dein Branch wurde in den 'gh-pages' Branch des Originals auf GitHub gemergt. Du kannst deinen Branch auch lokal mergen.

Zuerst wechsle in den Branch, in den du mergen möchtest—in diesem Fall den Branch 'gh-pages'.

git checkout gh-pages

Sage jetzt Git, welchen Branch du mergen möchtest—in diesem Fall deinen Feature-Branch, der mit "add-" beginnt.

git merge <BRANCHNAME>

Räume auf, indem du deinen Feature-Branch löschst. Jetzt, da er gemergt wurde, brauchst du ihn nicht wirklich mehr.

git branch -d <BRANCHNAME>

Du kannst den Branch auch von deinem Remote auf GitHub löschen:

git push <REMOTENAME> --delete <BRANCHNAME>

Vom Upstream pullen

Und last but not least, das Original hat sich seit dem Merge deines Pull Requests geändert—Reporobot hat deinen Namen zur Website hinzugefügt! Falls du diese Änderungen vom Upstream pullst, wirst du auf dem neuesten Stand sein und auch diese Version haben. Du kannst es tatsächlich auch live sehen unter: yourusername.github.io/patchwork.

Um vom ursprünglichen Upstream zu pullen:

git pull upstream gh-pages
