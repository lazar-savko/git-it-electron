Pull Requests

Oft, wenn du Änderungen und Verbesserungen an einem Projekt machst, das du geforkt hast, wirst du diese Änderungen an den Maintainer des Originals senden und anfragen, dass sie diese Änderungen in das Original pullen, damit alle von den Updates profitieren können—das ist ein Pull Request.

Wir möchten dich zur Liste der Workshop-Absolventen hinzufügen, also erstelle einen Pull Request, um deine Benutzername-Datei zum Original hinzuzufügen: github.com/jlord/patchwork.

Einen Pull Request erstellen

Besuche das ursprüngliche Repository, das du geforkt hast, auf GitHub, in diesem Fall http://github.com/jlord/patchwork.

Oft wird GitHub erkennen, wenn du einen Branch zu einem Fork gepusht hast und es oben auf der Website des Originals anzeigen. Falls du das mit deinem 'add-username' Branch siehst, kannst du klicken, um einen Pull Request von dort zu erstellen. Falls nicht:

  • Klicke auf den grünen 'New pull request' Button.
  • Wähle den Branch mit den Änderungen aus, die du einreichen möchtest. Es sollte derjenige mit 'add-yourusername' sein.

Du wirst jetzt eine Seite mit den Details des Pull Requests sehen, den du gerade einreichst. Diese Seite zeigt die Commits und Änderungen in Form eines Diffs, die mit deinem Pull Request verbunden sind, verglichen mit dem 'gh-pages' Branch des Originals.

Falls das ursprüngliche Repository eine Beitragsdokumentation hat, wird GitHub darauf verlinken. Das ist Dokumentation von Repository-Besitzern darüber, wie man am besten Beiträge zu diesem Projekt leistet—sehr hilfreich zu lesen, falls du möchtest, dass deine Änderungen übernommen werden!

Falls alles auf der Seite gut aussieht und wie du es erwartest:

  • Klicke auf 'Create pull request'
  • Füge einen Titel und eine Beschreibung zu den Änderungen hinzu, die du vorschlägst, dass der ursprüngliche Autor einholt.
  • Klicke auf 'Send pull request'!

Bingo! Du hast einen Pull Request eingereicht—nimm dir ein paar Sekunden, um den Moment zu genießen.

Falls alles mit deinem Pull Request in Ordnung ist, wird er innerhalb von Momenten gemergt. Falls er nicht automatisch innerhalb von ein paar Minuten gemergt wird, wirst du wahrscheinlich einige Kommentare von @reporobot erhalten, warum er ihn nicht mergen konnte. Falls das der Fall ist, schließe deinen Pull Request auf GitHub, mache die notwendigen Änderungen zu deinem Branch, pushe diese Änderungen zurück zu deinem Fork und öffne ihn wieder (das wird @reporobot dazu bringen, ihn nochmal zu überprüfen).
