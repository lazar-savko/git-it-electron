Füge einen Branch lokal zu deinem geforkten Repository hinzu, um an deinen Änderungen zu arbeiten.

Branches

Git-Repositories verwenden Branches, um Arbeit bei Bedarf zu isolieren. Es ist übliche Praxis, beim Arbeiten an einem Projekt oder mit anderen an einem Projekt einen Branch zu erstellen, um deine Arbeitsänderungen zu speichern. So kannst du deine Arbeit machen, während der Hauptbranch, üblicherweise 'main' genannt, stabil bleibt. Wenn die Arbeit an deinem Branch abgeschlossen ist, mergest du ihn zurück in den 'main' Branch.

Das Diagramm unten zeigt, wie du von deinem 'main' Branch abzweigen, arbeiten und dann diese Änderungen zurück in 'main' mergen kannst. Du kannst sogar von einem Branch abzweigen, wenn du es brauchst, der 'main' Branch muss nicht die Basis sein.

Für eine großartige Visualisierung, wie Branches in einem Projekt funktionieren, siehe diesen GitHub Guide: guides.github.com/overviews/flow

GitHub Pages

GitHub wird automatisch statische Website-Dateien in Branches namens 'gh-pages' bereitstellen und hosten. Dieser kostenlose Service heißt GitHub Pages. Da das Projekt, das du geforkt hast, eine Website erstellt, ist sein Hauptbranch 'gh-pages' genannt anstatt 'main'. Alle Repositories, die einen 'gh-pages' Branch mit Website Dateien haben, können live online mit diesem URL-Muster gefunden werden:

http://githubusername.github.io/repositoryname

Einen Branch erstellen

Wenn du einen Branch erstellst, kopiert Git alles vom aktuellen Branch, auf dem du dich befindest, und platziert es in dem Branch, den du erstellt haben möchtest.

Während du dich noch innerhalb deines lokalen 'patchwork' Repositories befindest, tippe git status, um zu sehen, auf welchem Branch du dich gerade befindest. Git sollte antworten, dass du dich auf dem 'gh-pages' Branch befindest.

Erstelle jetzt einen neuen Branch und nenne ihn "add-<username>", wobei 'username' dein Benutzername ist. Zum Beispiel "add-jlord". Branches sind case-sensitive, also benenne deinen Branch genau so, wie dein GitHub-Name erscheint.

git branch <BRANCHNAME>

Jetzt hast du einen Branch mit einem neuen Namen, der identisch mit 'gh-pages' ist.

Um in diesen Branch zu gehen und daran zu arbeiten, checkst du einen Branch aus. Gehe auf deinen neuen Branch:

git checkout <BRANCHNAME>

Schritt: Eine neue Datei erstellen

Zurück in deinem Texteditor:

  • Erstelle eine neue Datei namens "add-<USERNAME>.txt", wobei 'username' dein Benutzername ist. Zum Beispiel "add-jlord.txt".
  • Schreibe dann einfach deinen GitHub-Benutzernamen hinein, das war's. Zum Beispiel würde ich 'jlord' tippen.
  • Speichere diese Datei im 'contributors' Ordner in Patchwork: Patchwork/contributors/add-yourusername.txt
  • Als nächstes, checke deine Änderungen ein (siehe unten).

Check-in

Gehe durch die Schritte für das Einchecken eines Projekts:

git status

git add <contributors/FILENAME>

git commit -m "commit message"

Pushe jetzt dein Update zu deinem Fork, 'origin', auf GitHub:

git push origin <BRANCHNAME>

Berechtigung verweigert...Fehler: 403

Du pushst Änderungen zu einem Repository, auf das du keinen Schreibzugriff hast. In diesem Fall pushst du wahrscheinlich zu dem ursprünglichen 'jlord/patchwork'. Stelle sicher, dass du zu 'origin' pushst und dass es auf die Adresse deines Forks auf GitHub zeigt. Um zu überprüfen und zu sehen, was deine Remotes sind und wohin sie zeigen, führe git remote -v aus. Du solltest 'upstream' haben, das auf 'jlord/patchwork' zeigt, und 'origin', das auf 'yourusername/patchwork' zeigt.

Um ein Remote zu reparieren, das auf den falschen Ort zeigt, kannst du seine URL neu setzen: git remote set-url origin .

Authentifizierung fehlgeschlagen...Fehler: 401

Deine Identität konnte nicht verifiziert werden. Du könntest 2FA (Two Factor Authentication) aktiviert haben, in diesem Fall musst du ein persönliches Zugriffstoken als dein Passwort verwenden. Du kannst eines generieren, indem du diesen Anweisungen folgst. Du musst dieses aufbewahren und anstelle deines Passworts verwenden, wenn du dazu aufgefordert wirst. Du kannst auch dieses Zugriffstoken auf deinem Computer speichern, damit du es nicht erneut eingeben musst.

Datei NICHT im contributors Ordner

Die Datei, die du erstellst, sollte in den bestehenden 'contributors' Ordner im Patchwork-Repository platziert werden. Falls du sie woanders platziert hast, verwende einfach Finder oder Windows Explorer, um deine Datei in den Ordner zu verschieben. Du kannst git status nochmal überprüfen und du wirst feststellen, dass es deine Änderungen sieht. Stage und committe dann "alle" (-A) dieser Änderungen (Hinzufügungen und Löschungen) mit den Befehlen unten.

git add -A

git commit -m "move file into contributors folder"

Branch-Name erwartet: _____

Der Branch-Name sollte genau deinem Benutzernamen entsprechen. Um deinen Branch-Namen zu ändern:

git branch -m <NEWBRANCHNAME>

Wenn du deine Updates gemacht hast, verifiziere nochmal!

  • Du kannst einen Branch erstellen und in einer Zeile zu ihm wechseln
  • git checkout -b <BRANCHNAME>
  • Einen neuen Branch erstellen
  • git branch <BRANCHNAME>
  • Auf einen Branch wechseln
  • git checkout <BRANCHNAME>
  • Die Branches auflisten
  • git branch
  • Einen Branch umbenennen, auf dem du dich gerade befindest
  • git branch -m <NEWBRANCHNAME>
  • Überprüfen, auf welchem Branch du arbeitest
  • git status
