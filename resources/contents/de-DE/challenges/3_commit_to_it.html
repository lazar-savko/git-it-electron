Erstelle eine Datei in deinem neuen Repository, füge etwas zu dieser Datei hinzu und committe diese Änderung mit Git.

Commits

Commits sind das Herzstück der Git-Nutzung. Sie sind die Momente, in denen du deine geleistete Arbeit speicherst und beschreibst. Sie sind die Markierungen in der Zeitleiste der Geschichte deines Projekts.

Eine neue Datei erstellen

Jetzt, da du ein Repository gestartet hast, lass uns eine Datei hinzufügen.

Öffne deinen Texteditor und erstelle eine neue leere Datei. Schreibe jetzt ein wenig Text, vielleicht tippe "Hallo!", und speichere die Datei als 'readme.txt' im 'hello-world' Ordner, den du in der letzten Challenge erstellt hast.

Status prüfen + Änderungen hinzufügen und committen

Prüfe als nächstes den Status deines Repositories, um herauszufinden, ob es Änderungen gegeben hat. Du weißt, dass du etwas geändert hast, aber weiß Git das?

Stelle sicher, dass du dich noch in deinem 'hello-world' Verzeichnis befindest, wenn du diese Befehle ausführst. Verwende Git, um zu sehen, was sich in deinem Repository geändert hat:

Zuerst, prüfe den Status:

git status

Git sollte dir sagen, dass eine Datei hinzugefügt wurde.

Dann füge die Datei, die du gerade erstellt hast, hinzu, damit sie Teil der Änderungen wird, die du committen (auch bekannt als speichern) wirst:

git add readme.txt

Schließlich committe diese Änderungen zur Geschichte des Repositories mit einer kurzen (m) Nachricht, die die Updates beschreibt.

git commit -m "Created readme"

Schritt: Weitere Änderungen vornehmen

Füge jetzt eine weitere Zeile zu 'readme.txt' hinzu und speichere die Datei erneut.

Im Terminal kannst du die Differenz zwischen der Datei jetzt und wie sie bei deinem letzten Commit war, anzeigen.

Sage Git, dir den Diff zu zeigen:

git diff

Jetzt mit dem, was du gerade oben gelernt hast, committe diese neueste Änderung.
