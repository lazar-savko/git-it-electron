Unten sind die Terminal- und Git-Befehle aufgelistet, die in den Git-it Challenges verwendet werden.

Bash/Terminal Grundlagen

  • Einen neuen Ordner erstellen (auch bekannt als make directory)
    • mkdir <FOLDERNAME>
  • In einen bestehenden Ordner navigieren (auch bekannt als change directory)
    • cd <FOLDERNAME>
  • Die Elemente in einem Ordner auflisten
    • ls

Git konfigurieren

  • Git-Version überprüfen
    • git --version
  • Deinen Namen setzen
    • git config --global user.name "Dein Name"
  • Deine E-Mail setzen
    • git config --global user.email deineemail@beispiel.com
  • Dein GitHub-Konto setzen (case sensitive)
    • git config --global user.username uSeRnAmE

Git Grundlagen

  • Git für einen Ordner aktivieren
    • git init
  • Status von Änderungen an einem Repository überprüfen
    • git status
  • Änderungen an Dateien anzeigen
    • git diff
  • Änderungen einer Datei zum Committen hinzufügen
    • git add <FILENAME>
  • Um alle Dateiänderungen hinzuzufügen
    • git add .
  • Um die hinzugefügten Änderungen zu committen (auch bekannt als speichern) mit einer kurzen Nachricht, die die Änderungen beschreibt
    • git commit -m "deine commit nachricht"
  • Ein Repository auf deinen Computer kopieren
    • git clone <URL>

Git Branches

  • Einen neuen Branch erstellen
    • git branch <BRANCHNAME>
  • Auf einen Branch wechseln
    • git checkout <BRANCHNAME>
  • Du kannst einen Branch erstellen und in einer Zeile zu ihm wechseln
    • git checkout -b <BRANCHNAME>
  • Die Branches auflisten
    • git branch
  • Einen Branch umbenennen, auf dem du dich gerade befindest
    • git branch -m <NEWBRANCHNAME>

Git Remotes

  • Remote-Verbindungen hinzufügen
    • git remote add <REMOTENAME> <URL>
  • Eine URL zu einem Remote setzen
    • git remote set-url <REMOTENAME> <URL>
  • Remote-Verbindungen anzeigen
    • git remote -v

Pullen in Git

  • Änderungen einholen
    • git pull
  • Änderungen von einem Remote-Branch einholen
    • git pull <REMOTENAME> <REMOTEBRANCH>
  • Änderungen am Remote sehen, bevor du sie einholst
    • git fetch --dry-run

Push & Merge in Git

  • Änderungen pushen
    • git push <REMOTENAME> <BRANCHNAME>
  • Einen Branch in den aktuellen Branch mergen
    • git merge <BRANCHNAME>

Remotes und Branches in Git löschen

  • Einen lokalen Branch löschen
    • git branch -D <BRANCHNAME>
  • Einen Remote-Branch löschen
    • git push <REMOTENAME> --delete <BRANCHNAME>
